60개 용어
LLM Large Language Model
⭐

거대한 자동완성기. 엄청난 텍스트를 학습해서 다음 단어를 예측하는 AI의 핵심 엔진.

Transformer 아키텍처(Vaswani et al., 2017) 기반. Self-attention: Q,K,V 행렬로 토큰 간 관계 가중치 계산. 파라미터 수: GPT-4 ~1.8T(추정), Claude 3 비공개. Pre-training(다음 토큰 예측) → RLHF/DPO로 정렬. Scaling law: 파라미터/데이터/컴퓨팅 동시 증가 시 로그 스케일 성능 향상.

토큰 Token
⭐

AI가 글을 읽는 단위. 한국어 "안녕하세요"는 3-5토큰. 토큰 수 = 비용.

BPE(Byte Pair Encoding) 또는 SentencePiece로 분할. 한국어는 영어 대비 2-3배 비효율(바이트당 정보량 차이). cl100k_base(GPT-4): 100K vocab. 1K 토큰 ≈ 영어 750단어, 한국어 300-400자. 비용: input/output 별도 과금, 캐시된 토큰은 할인.

컨텍스트 윈도우 Context Window
⭐

AI의 단기 기억력 크기. 창문이 클수록 더 많은 자료를 동시에 볼 수 있다.

Transformer의 self-attention은 O(n²) 복잡도 → 긴 컨텍스트 비용 급증. 해결: Sparse attention, Ring attention, 슬라이딩 윈도우. Claude 1M, Gemini 2M은 특수 아키텍처. "Lost in the middle" 문제: 긴 컨텍스트 중간 정보 검색 성능 저하(Liu et al., 2023).

멀티모달 Multimodal
⭐

글뿐 아니라 사진, 소리, 영상도 이해하는 AI. 사람처럼 여러 감각을 가진 것.

Vision encoder(ViT) + LLM 결합. 이미지를 패치로 분할 → 임베딩 → LLM 입력에 concat. GPT-4o: 네이티브 멀티모달(별도 인코더 없이 통합 학습). Cross-attention vs early fusion vs late fusion 아키텍처 선택.

환각 Hallucination
⭐

AI가 자신있게 거짓말하는 것. "서울의 수도는 부산입니다"처럼 그럴듯하지만 틀린 답.

원인: (1) 학습 데이터 편향, (2) 확률적 생성의 본질(beam search vs sampling), (3) knowledge cutoff 이후 정보 부재. 대응: RAG로 외부 사실 주입, self-consistency(여러 답변 교차 검증), 불확실성 표현 학습. 완전 제거는 이론적으로 불가능(확률 모델의 한계).

파인튜닝 Fine-tuning
⭐⭐

범용 AI를 특정 분야 전문가로 만드는 추가 학습. 의대 졸업 후 전문의 과정.

Full fine-tuning: 전체 파라미터 업데이트, 수십 GB VRAM 필요. SFT(Supervised Fine-Tuning): labeled 데이터로 지도 학습. DPO(Direct Preference Optimization): RLHF 대비 간소화. Catastrophic forgetting: 새 데이터 학습 시 기존 지식 손실 위험.

LoRA / PEFT Low-Rank Adaptation
⭐⭐⭐

모델 전체를 바꾸지 않고 일부만 살짝 조정하는 경량 튜닝. 전체 리모델링 대신 인테리어만 바꾸는 것.

LoRA(Hu et al., 2021): 가중치 행렬 W에 저랭크 분해 ΔW = BA (B: d×r, A: r×k, r≪d). 학습 파라미터 0.1-1%. QLoRA: 4-bit 양자화 + LoRA로 단일 GPU 파인튜닝. Rank r=8-64가 일반적. 어댑터 합성(adapter merging)으로 여러 LoRA 결합 가능.

MoE Mixture of Experts
⭐⭐⭐

여러 전문가 중 상황에 맞는 2-3명만 출동하는 시스템. 전체가 일하는 것보다 효율적.

Shazeer et al.(2017) → Mixtral, DeepSeek-V3. Router network가 토큰별로 top-k expert 선택(보통 k=2). 장점: 총 파라미터 크지만 활성 파라미터 작아 추론 효율적. DeepSeek-V3: 671B total, 37B active. Load balancing loss로 expert 균등 활용. Auxiliary loss 없는 방법도 연구 중.

임베딩 Embedding
⭐⭐

단어나 문장을 숫자 좌표로 변환. "왕-남자+여자=여왕"처럼 의미를 수학으로 계산 가능하게 만드는 것.

Dense vector (768-3072 dims). 학습: contrastive learning(positive/negative pair). 모델: OpenAI text-embedding-3, Cohere embed-v3, BGE-M3(다국어). 유사도: cosine similarity가 표준. ANN 검색: HNSW, IVF. 한국어: multilingual 모델 또는 한국어 특화(KoSimCSE) 사용.

추론 / CoT Reasoning / Chain-of-Thought
⭐⭐ 2025+

AI가 바로 답하지 않고 "음... 먼저 이걸 생각하고, 그 다음..."처럼 단계별로 사고하는 것.

Wei et al.(2022). "Let's think step by step"으로 수학/논리 성능 급증. o1(2024.09): hidden CoT + RL로 추론 학습. DeepSeek R1: $6M으로 o1급 성능, MIT 공개. Process reward model: 각 단계별 보상. Test-time compute scaling: 추론 시 더 많은 연산 → 더 나은 결과. Thinking budget 조절 가능.

증류 Distillation
⭐⭐⭐

대학교수의 지식을 고등학생에게 전수하는 것. 작은 모델이 큰 모델의 실력을 물려받음.

Hinton et al.(2015). Teacher-Student 프레임워크. Soft label(확률 분포)로 dark knowledge 전달. Temperature scaling으로 분포 평탄화. DeepSeek R1-Distill: R1 추론 능력을 Qwen/Llama 기반 소형 모델에 전달. 2단계: SFT on teacher output → RL fine-tuning.

모델 캐스케이딩 Model Cascading
⭐⭐ 2025+

쉬운 질문은 싼 AI, 어려운 질문은 비싼 AI에게. 동네병원 → 대학병원 의뢰 시스템.

Router가 쿼리 난이도 판단 → 적절한 모델 라우팅. Confidence-based: 작은 모델이 불확실하면 escalate. OpenClaw: 서브에이전트에 저렴한 모델, 최종 판단에 Opus. FrugalGPT(Chen et al., 2023): API 비용 최대 98% 절감. 실전: Sonnet(근육) + Opus(두뇌) 패턴.

프롬프트 캐싱 Prompt Caching
⭐⭐ 2025+

매번 같은 설명을 반복하는 대신 "아까 그거"로 넘어가는 것. 시간과 비용 절약.

Anthropic(2024.08): 시스템 프롬프트의 KV cache 재사용. TTL 5분(Anthropic) 또는 세션 단위. 비용: 캐시 write 1.25x, read 0.1x → 반복 호출 시 최대 90% 절감. OpenClaw: heartbeat 주기를 TTL보다 짧게 설정하면 캐시 상시 유지. OpenAI도 유사 기능 도입.

온디바이스 AI On-device AI
⭐

인터넷 없이 폰/PC에서 직접 돌아가는 AI. 개인정보가 밖으로 안 나가서 안전.

양자화(INT4/INT8)로 모델 축소. NPU(Neural Processing Unit) 활용: Apple Neural Engine, Qualcomm Hexagon, Samsung Exynos. Galaxy AI: Gemini Nano 온디바이스 + 클라우드 하이브리드. Apple Intelligence: 3B 모델 온디바이스, Private Cloud Compute(서버측도 E2E 암호화). 한계: 모델 크기 제약(3-7B), 추론 속도.

MCP Model Context Protocol
⭐⭐ 2025+

AI 도구 연결의 USB-C. 예전엔 도구마다 다른 방식이었는데, 이제 하나의 표준으로 통일.

Anthropic 2024.11 공개. JSON-RPC 2.0 기반. Host→Client→Server 구조. 3요소: Resources(읽기전용), Tools(실행가능), Prompts(템플릿). 전송: stdio(로컬) 또는 HTTP+SSE(원격). Capability negotiation으로 보안. 채택: Cursor, Claude Desktop, Zed, Sourcegraph. 비판: Peter Steinberger "CLI로 충분", 실제 MCP 서버 품질 편차 큼. 표준은 맞지만 만능은 아님.

함수 호출 Function Calling
⭐⭐

AI한테 "계산기 써", "날씨 검색해"처럼 외부 도구를 쓸 수 있게 해주는 기능.

JSON Schema로 함수 시그니처 정의 → LLM이 structured output으로 호출 결정 → 호스트가 실행 → 결과를 다시 LLM에 전달. OpenAI(2023.06) 최초 도입. Parallel function calling: 여러 도구 동시 호출. MCP가 이를 표준화한 상위 개념.

A2A Agent-to-Agent Protocol
⭐⭐⭐ 2025+

AI들끼리 대화하는 프로토콜. "너는 번역 담당, 나는 요약 담당" 하고 협업.

Google 2025.04 제안. Agent Card(JSON): 에이전트 능력 선언. Task lifecycle: submitted→working→done/failed. Streaming SSE. MCP와 상호보완(MCP=도구, A2A=에이전트간). RC v1.0이지만 실제 클라이언트/서버 구현 거의 없음(2026.02). 사실상 사양서만 존재.

스킬 Skills
⭐⭐ 2025+

AI에게 새 능력을 가르치는 플러그인. "날씨 스킬 설치하면 날씨를 알려줘".

OpenClaw: SKILL.md(마크다운 지시문) + 스크립트/설정 zip. 설치 시 에이전트가 description 읽고 자동 매칭. ClawHub: 커뮤니티 공유. Claude Code: skills.sh로 프로젝트별 자동 실행 스크립트. 보안 위험: 악성 스킬이 파일시스템/네트워크 접근 가능 → 신뢰 검증 필수.

도구 Tools
⭐

AI가 쓸 수 있는 연장통. 파일 읽기, 웹 검색, 코드 실행 등 각각이 "도구".

Tool 정의: name + description + input schema(JSON Schema). LLM이 도구 선택/파라미터 결정. Tool result → assistant message로 피드백. 도구 수 증가 시 선택 정확도 저하 → tool routing, tool profiles로 대응. OpenClaw: read/write/exec/browser/message 등 15+ 내장.

RAG Retrieval-Augmented Generation
⭐⭐

AI에게 오픈북 시험을 치르게 하는 것. 답하기 전에 자료를 찾아 읽으니 틀릴 확률 감소.

Lewis et al.(2020). Query→Embedding→Vector DB(Top-K)→Context injection→생성. Dense retrieval: bi-encoder(DPR). Chunking 전략: fixed(512 tokens), semantic, recursive. 2025: Anthropic Contextual Retrieval(요약 검색→원문), Long-context(1M)가 RAG 대체 가능성 논쟁. Hybrid: sparse(BM25)+dense 결합이 실전 최적.

지식 베이스 Knowledge Base
⭐

AI의 참고서. 회사 매뉴얼, 제품 정보 등을 정리해둔 저장소.

구조: 문서→청킹→임베딩→벡터 DB 저장. 업데이트 파이프라인: 문서 변경 감지→재인덱싱. 메타데이터 필터링으로 검색 정밀도 향상. 기업용: Confluence/Notion 연동. OpenClaw: knowledge/ 디렉토리에 마크다운으로 관리, memory_search로 시맨틱 검색.

벡터 DB Vector Database
⭐⭐

일반 DB가 숫자/텍스트를 저장한다면, 벡터 DB는 "의미"를 저장. "비슷한 뜻" 검색 가능.

ANN(Approximate Nearest Neighbor) 알고리즘: HNSW(Hierarchical Navigable Small World), IVF(Inverted File Index), PQ(Product Quantization). 제품: Pinecone(관리형), Weaviate(오픈소스), ChromaDB(경량), pgvector(PostgreSQL 확장). 벤치마크: QPS vs recall tradeoff. 필터링: pre-filter vs post-filter.

에이전틱 AI Agentic AI
⭐ 2025+

"대답하는 AI"에서 "행동하는 AI"로. 질문에 답하는 게 아니라 목표를 주면 알아서 계획→실행→검증.

ReAct(Yao et al., 2022): Reasoning+Acting 결합. 루프: Observe→Think→Act→Observe. 도구 사용이 핵심 차이. 2025년 전환점: Claude 3.5의 Computer Use, OpenAI의 Operator. 에이전트 ≠ 챗봇: 챗봇은 1턴 응답, 에이전트는 multi-turn + tool + memory + planning.

에이전트 Agent
⭐

자율적으로 도구를 써서 일하는 AI 직원. 지시하면 계획 세우고 실행까지.

구성요소: LLM(두뇌) + Tools(손) + Memory(기억) + Planning(계획). 실행 패턴: single-agent loop, multi-agent delegation, hierarchical. 상태: stateless(매 턴 독립) vs stateful(세션 유지). OpenClaw/Claude Code는 stateful agent의 대표.

하네스 Harness
⭐⭐ 2025+

말(LLM)에 채우는 마구(고삐+안장). 모델의 힘을 제어하고 유용하게 만드는 제어 장치.

모델 위의 오케스트레이션 레이어. 포함: 도구 라우팅, 컨텍스트 관리, 메모리 시스템, 스케줄링(cron/heartbeat), 보안(guardrail/sandbox), 채널 연결. OpenClaw = 가장 포괄적 하네스. "모델만 좋아진 게 아니라 하네스가 성숙했다"가 2025→2026 핵심 변화.

오케스트레이터 Orchestrator
⭐⭐

여러 AI를 지휘하는 지휘자. 누구한테 무슨 일 시킬지 정하고, 결과 모아서 정리.

패턴: (1) Router(쿼리 분류→전문 에이전트), (2) Planner(작업 분해→병렬/순차 실행), (3) Supervisor(실행 모니터링+재시도). LangGraph, CrewAI, AutoGen 등 프레임워크. OpenClaw: 메인 에이전트가 sessions_spawn으로 서브에이전트 생성하는 내장 오케스트레이션.

서브에이전트 Sub-agent
⭐⭐ 2025+

메인 AI가 "이거 조사해와"하고 보내는 부하 AI. 독립적으로 일하고 결과 보고.

격리된 세션에서 실행(isolated session). 부모 컨텍스트 상속 안 함 → 토큰 절약 + 병렬화. 결과는 push 기반 알림. 모델 라우팅: 서브에이전트에 저렴한 모델 할당 가능. 한계: 20-30KB 출력 제한, 부모와 정보 격차, 팩트체크 필수.

에이전트 팀 Agent Teams
⭐⭐⭐ 2025+

AI 팀 프로젝트. 한 명은 코딩, 한 명은 테스트, 한 명은 문서화... 동시에 일함.

Claude Code Agent Teams(2026.02): /agents로 병렬 서브에이전트 실행. Cursor Self-Driving: 다수 에이전트가 자율적 커밋. GitHub Agent HQ: 멀티 에이전트 거버넌스. 핵심 과제: 충돌 해결(git merge), 일관성 유지, 비용 제어.

스웜 Swarm
⭐⭐⭐

개미 군단처럼 많은 AI가 각자 단순 규칙을 따르면서 전체적으로 복잡한 일을 해결.

OpenAI Swarm SDK(2024.10): 경량 에이전트 핸드오프 프레임워크. Agent 정의(instructions+functions) → handoff로 에이전트 전환. 실험적/교육용. 프로덕션에서는 Agents SDK로 진화. 분산 시스템의 consensus 문제와 유사한 과제.

메모리 Memory
⭐⭐ 2025+

AI의 기억력. 대화 내용, 사용자 취향, 배운 것을 저장해서 다음에 기억.

단기: 컨텍스트 윈도우 내 대화 히스토리. 장기: 파일 기반(MEMORY.md), 벡터 검색(memory_search). OpenClaw: BM25+벡터+리랭킹(QMD 백엔드). 시간 감쇠: 최신 기억 우선. Compaction: 오래된 대화 요약→압축. 과제: 무엇을 기억하고 무엇을 잊을지 판단.

하트비트 Heartbeat
⭐⭐ 2025+

AI가 주기적으로 깨어나 "할 일 있나?" 확인하는 알람. 비서가 30분마다 체크하는 것.

OpenClaw: 설정 주기(기본 30분)마다 HEARTBEAT.md 읽기 → 체크리스트 실행. 빈 파일이면 API 콜 스킵(토큰 절약). 프롬프트 캐싱과 연계: TTL보다 짧은 주기로 캐시 워밍. Cron과 차이: heartbeat는 메인 세션 컨텍스트 유지, cron은 격리 세션.

RLHF Reinforcement Learning from Human Feedback
⭐⭐⭐

사람이 "이 답이 더 좋아"라고 AI에게 피드백하면서 가르치는 방법.

Christiano et al.(2017). 3단계: (1) SFT 기본 학습, (2) Reward model 학습(인간 선호 데이터), (3) PPO/GRPO로 RL 최적화. 대안: DPO(Direct Preference Optimization) — reward model 없이 직접 최적화. RLAIF: 인간 대신 AI가 피드백. Constitutional AI(Anthropic): 원칙 기반 자기 개선.

Anthropic(2022). 원칙 목록(helpfulness, harmlessness, honesty) → AI가 자기 출력을 원칙에 비춰 수정 → 수정된 데이터로 RL. Red Teaming 자동화: AI가 자기 취약점 탐색. 장점: 인간 피드백 의존도 감소, 확장성. Claude의 안전성 기반 기술.

치명적 삼중주 Lethal Trifecta
⭐⭐ 2025+

AI 보안 사고가 일어나는 3가지 조합: 비밀 데이터 + 외부 도구 + 신뢰할 수 없는 입력. 셋 다 있으면 사고 필연.

Simon Willison 명명. (1) Access to private data(이메일, 파일), (2) Ability to take actions(이메일 전송, 코드 실행), (3) Exposure to untrusted input(웹 콘텐츠, 사용자 메시지). 에이전트 시대에 3개 동시 충족이 기본 → 보안 필수. 대응: 최소 권한 원칙, 행동 전 확인, 입력 sanitization.

CaMeL Capability-aware ML
⭐⭐⭐ 2025+

DeepMind(2025). Capability-aware ML. 도구 호출을 DAG(Directed Acyclic Graph)로 분리: 데이터 흐름 추적 가능. Taint tracking: 외부 데이터가 어떤 행동에 영향 미쳤는지 추적. 프롬프트 인젝션 대응: 오염된 데이터의 행동 차단. 아직 실험 단계, 프로덕션 구현 거의 없음.
